1. Introdução

Este documento fornece instruções detalhadas para instalar, configurar e implantar o sistema de airdrop WebyteCoin ($WBC). O sistema permite que os usuários participem do airdrop completando tarefas específicas e reivindicando tokens $WBC.

Sobre o WebyteCoin Airdrop

O WebyteCoin Airdrop é uma plataforma completa para distribuição de tokens $WBC através de um sistema de tarefas e referências. Os usuários podem conectar suas carteiras, completar tarefas como seguir perfis em redes sociais, participar de grupos de Telegram, e receber tokens como recompensa.

O sistema inclui as seguintes funcionalidades principais:

  • Conexão de carteira Web3 (MetaMask, WalletConnect, etc.)
  • Sistema de verificação de tarefas (Twitter, Telegram, etc.)
  • Rastreamento de progresso com persistência de dados
  • Sistema de referência para recompensar usuários que convidam outros
  • Verificação de elegibilidade para o airdrop
  • Reivindicação de tokens com feedback visual
  • Suporte para múltiplas redes blockchain

2. Requisitos do Sistema

Antes de iniciar a instalação, certifique-se de que seu ambiente atende aos seguintes requisitos:

Requisitos de Hardware
  • Servidor: VPS com pelo menos 2GB de RAM e 1 vCPU
  • Recomendado: 4GB RAM, 2 vCPU para melhor desempenho
  • Armazenamento: Mínimo de 20GB de espaço em disco
Requisitos de Software
  • Sistema Operacional: Ubuntu 20.04 LTS ou superior
  • Node.js: Versão 16.x ou superior
  • Servidor Web: Nginx ou Apache
  • Certificado SSL: Let's Encrypt ou similar
  • Banco de Dados: MongoDB ou PostgreSQL (opcional)
  • Redis: Para cache e gerenciamento de sessões (opcional)
Requisitos de Rede
  • Domínio: Um domínio registrado para hospedar a aplicação
  • DNS: Configuração de registros A ou CNAME apontando para o servidor
  • Portas: 80 (HTTP), 443 (HTTPS), 22 (SSH)
  • Firewall: Configurado para permitir tráfego nas portas necessárias
Requisitos de Blockchain
  • Conta na Infura ou Alchemy: Para acesso às redes blockchain
  • Carteira Ethereum: Com fundos suficientes para implantação de contratos
  • MetaMask: Ou outra carteira compatível com Web3 para implantação
  • Tokens $WBC: Para distribuição através do contrato de airdrop

# Ativar o site

Configuração do Firewall

Configure o firewall UFW para proteger seu servidor:

# Instalar UFW se não estiver instalado
sudo apt install -y ufw

# Configurar regras básicas
sudo ufw default deny incoming
sudo ufw default allow outgoing
sudo ufw allow ssh
sudo ufw allow http
sudo ufw allow https

# Ativar o firewall
sudo ufw enable

# Verificar status
sudo ufw status

Atenção

Certifique-se de que a porta SSH (22) está permitida antes de ativar o firewall, caso contrário, você poderá perder o acesso ao servidor.

4. Configuração do Contrato

O sistema de airdrop WebyteCoin requer a implantação de um contrato inteligente para gerenciar a distribuição de tokens.

Preparação do Contrato

O contrato inteligente WebyteCoinAirdrop.sol deve ser configurado com os parâmetros corretos:

// SPDX-License-Identifier: MIT
pragma solidity ^0.8.17;

import "@openzeppelin/contracts/token/ERC20/IERC20.sol";
import "@openzeppelin/contracts/access/Ownable.sol";
import "@openzeppelin/contracts/security/ReentrancyGuard.sol";

contract WebyteCoinAirdrop is Ownable, ReentrancyGuard {
    IERC20 public token;
    
    // Valor de tokens por usuário
    uint256 public airdropAmount = 1000 * 10**18; // 1000 tokens por usuário
    
    // Bônus de referral
    uint256 public referralBonus = 200 * 10**18; // 200 tokens por referral
    
    // Períodos de início e término
    uint256 public startTime;
    uint256 public endTime;
    
    // Status do airdrop
    bool public airdropActive = false;
    
    // Mapeamento para rastrear reivindicações
    mapping(address => bool) public hasClaimed;
    
    // Mapeamento para rastrear referências
    mapping(address => address) public referrals;
    mapping(address => uint256) public referralCount;
    
    // Eventos
    event TokensClaimed(address indexed user, uint256 amount);
    event ReferralBonusPaid(address indexed referrer, address indexed referred, uint256 amount);
    
    constructor(address _tokenAddress) {
        token = IERC20(_tokenAddress);
    }
    
    // Configurar períodos de início e término
    function setAirdropTime(uint256 _startTime, uint256 _endTime) external onlyOwner {
        require(_startTime < _endTime, "Start time must be before end time");
        startTime = _startTime;
        endTime = _endTime;
    }
    
    // Ativar/desativar airdrop
    function toggleAirdropStatus() external onlyOwner {
        airdropActive = !airdropActive;
    }
    
    // Verificar elegibilidade
    function isEligible(address _user) public view returns (bool) {
        return airdropActive && 
               block.timestamp >= startTime && 
               block.timestamp <= endTime && 
               !hasClaimed[_user];
    }
    
    // Reivindicar tokens
    function claimTokens() external nonReentrant {
        require(isEligible(msg.sender), "Not eligible to claim");
        
        hasClaimed[msg.sender] = true;
        token.transfer(msg.sender, airdropAmount);
        
        emit TokensClaimed(msg.sender, airdropAmount);
    }
    
    // Reivindicar com referência
    function claimWithReferral(address _referrer) external nonReentrant {
        require(isEligible(msg.sender), "Not eligible to claim");
        require(_referrer != msg.sender, "Cannot refer yourself");
        require(_referrer != address(0), "Invalid referrer address");
        
        hasClaimed[msg.sender] = true;
        referrals[msg.sender] = _referrer;
        referralCount[_referrer]++;
        
        token.transfer(msg.sender, airdropAmount);
        emit TokensClaimed(msg.sender, airdropAmount);
        
        // Pagar bônus de referência
        if (token.balanceOf(address(this)) >= referralBonus) {
            token.transfer(_referrer, referralBonus);
            emit ReferralBonusPaid(_referrer, msg.sender, referralBonus);
        }
    }
    
    // Resgatar tokens não utilizados (apenas proprietário)
    function withdrawTokens() external onlyOwner {
        uint256 balance = token.balanceOf(address(this));
        require(balance > 0, "No tokens to withdraw");
        token.transfer(owner(), balance);
    }
}
Compilação e Implantação

Use o Remix IDE para compilar e implantar o contrato:

  1. Acesse o Remix IDE
  2. Crie um novo arquivo e cole o código do contrato WebyteCoinAirdrop.sol
  3. Compile o contrato (Solidity Compiler 0.8.17+)
  4. Vá para a aba "Deploy & Run Transactions"
  5. Selecione "Injected Web3" como ambiente (certifique-se de que o MetaMask está conectado)
  6. Selecione a rede desejada no MetaMask (Ethereum, BSC, Polygon, etc.)
  7. Implante o contrato com o endereço do token ERC-20 como parâmetro
  8. Anote o endereço do contrato implantado
Configuração Pós-Implantação

Após a implantação, configure o contrato:

  1. Chame a função setAirdropTime para definir as datas de início e término
  2. Chame a função toggleAirdropStatus para ativar o airdrop
  3. Transfira tokens para o contrato (suficientes para cobrir todos os participantes esperados)
  4. Verifique o contrato no explorador de blocos (Etherscan, BscScan, etc.)

Dica

Implante o contrato em redes de teste (Goerli, BSC Testnet) primeiro para verificar se tudo está funcionando corretamente antes de implantar em redes principais.

7. Sistema de Reivindicação

O sistema de reivindicação é uma parte fundamental do airdrop WebyteCoin. Ele permite que os usuários completem tarefas e reivindiquem seus tokens.

Verificação de Tarefas

O sistema de verificação de tarefas inclui:

  • Verificação de seguidores no Twitter
  • Verificação de participação em grupos do Telegram
  • Verificação de retweets e curtidas
  • Verificação de participação em servidores Discord

Cada tarefa é verificada através de APIs ou simulação, dependendo da configuração.

Persistência de Dados

O sistema utiliza localStorage para persistir o progresso do usuário entre sessões. Para uma solução mais robusta, você pode implementar um banco de dados:

A implementação atual usa localStorage para persistência:

// Salvar progresso
const saveProgress = (tasks) => {
  localStorage.setItem('webytecoin-tasks', JSON.stringify(tasks));
};

// Carregar progresso
const loadProgress = () => {
  const savedTasks = localStorage.getItem('webytecoin-tasks');
  return savedTasks ? JSON.parse(savedTasks) : null;
};

Para implementar com MongoDB:

// Instalar dependências
// npm install mongodb

// Configurar conexão
import { MongoClient } from 'mongodb';

const uri = process.env.MONGODB_URI;
const client = new MongoClient(uri);

// Salvar progresso
async function saveProgress(address, tasks) {
  await client.connect();
  const database = client.db('webytecoin');
  const collection = database.collection('user-tasks');
  
  await collection.updateOne(
    { address },
    { $set: { tasks, updatedAt: new Date() } },
    { upsert: true }
  );
}

// Carregar progresso
async function loadProgress(address) {
  await client.connect();
  const database = client.db('webytecoin');
  const collection = database.collection('user-tasks');
  
  const result = await collection.findOne({ address });
  return result ? result.tasks : null;
}

Para implementar com PostgreSQL:

// Instalar dependências
// npm install pg

// Configurar conexão
import { Pool } from 'pg';

const pool = new Pool({
  connectionString: process.env.DATABASE_URL,
});

// Criar tabela
async function createTable() {
  await pool.query(`
    CREATE TABLE IF NOT EXISTS user_tasks (
      address VARCHAR(42) PRIMARY KEY,
      tasks JSONB NOT NULL,
      created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
      updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
    )
  `);
}

// Salvar progresso
async function saveProgress(address, tasks) {
  await pool.query(
    `INSERT INTO user_tasks (address, tasks) 
     VALUES ($1, $2)
     ON CONFLICT (address) 
     DO UPDATE SET tasks = $2, updated_at = CURRENT_TIMESTAMP`,
    [address, JSON.stringify(tasks)]
  );
}

// Carregar progresso
async function loadProgress(address) {
  const result = await pool.query(
    'SELECT tasks FROM user_tasks WHERE address = $1',
    [address]
  );
  
  return result.rows.length > 0 ? result.rows[0].tasks : null;
}
Sistema de Referência

O sistema de referência permite que os usuários convidem outros e recebam bônus:

  1. Cada usuário recebe um link de referência único baseado em seu endereço de carteira
  2. Quando um novo usuário se registra usando o link de referência, o referenciador é registrado
  3. Quando o novo usuário reivindica tokens, o referenciador recebe um bônus

O bônus de referência é configurado no contrato inteligente e pode ser ajustado pelo proprietário.

Dica

Para evitar fraudes, implemente verificações adicionais, como verificar se o endereço IP do usuário já participou do airdrop ou limitar o número de referências por endereço.

10. Solução de Problemas

Aqui estão algumas soluções para problemas comuns:

Problemas de Conexão de Carteira
  • Problema: Usuários não conseguem conectar suas carteiras
  • Solução: Verifique se as redes estão configuradas corretamente em lib/networks.ts
  • Solução: Certifique-se de que as chaves da Infura/Alchemy estão corretas
  • Solução: Verifique se o navegador do usuário tem extensões de carteira instaladas
Problemas de Verificação de Tarefas
  • Problema: As tarefas não são verificadas corretamente
  • Solução: Verifique os logs do servidor para erros de API
  • Solução: Certifique-se de que as APIs sociais estão funcionando
  • Solução: Verifique se as configurações de tarefas em config/tasks.json estão corretas
Problemas de Reivindicação de Tokens
  • Problema: Usuários não conseguem reivindicar tokens
  • Solução: Verifique se o contrato tem tokens suficientes
  • Solução: Certifique-se de que o airdrop está ativo no contrato
  • Solução: Verifique se o usuário já reivindicou tokens antes
  • Solução: Verifique se o usuário completou todas as tarefas obrigatórias
Problemas de Servidor
  • Problema: A aplicação não inicia
  • Solução: Verifique os logs do PM2: pm2 logs webytecoin
  • Solução: Verifique se todas as dependências estão instaladas
  • Solução: Verifique se o Node.js está na versão correta

Suporte

Se você encontrar problemas que não consegue resolver, entre em contato com o suporte técnico ou consulte a documentação adicional.
